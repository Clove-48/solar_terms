# 产品概述：黄道手札 · Ecliptic Notes

> 项目代号：二十四节气·天文科普沉浸式H5
> 最后更新：2026-06-14 | 开发状态：Phase 4 完成，Phase 5 验收中

## 1. 产品定位

### 1.1 一句话概括

一款以**手势交互为核心**、面向**青少年科普**的二十四节气沉浸式H5作品，以太阳黄经为科学主线，通过国风+科技融合的视觉语言，让用户在挥手间"唤醒"节气、理解节气背后的天文科学原理。

### 1.2 核心创新点

> 以太阳黄经为科学主线，通过手势交互"唤醒"二十四节气，呈现每个节气在黄道上的精确位置（太阳黄经度数），并用可视化方式解释节气背后的天文观测原理（圭表测影、日影变化等），让用户在沉浸式互动中建立"节气=科学"的认知。

### 1.3 核心价值

| 维度 | 内容 |
|------|------|
| **科学内核** | 以太阳黄经0°~360°为主轴，每15°一个节气，动态黄道带可视化 |
| **交互创新** | 基于TensorFlow.js Handpose的手势唤醒 + 7种降级方案确保普及性 |
| **科普定位** | 面向青少年，通俗易懂，手势+动画直观解释天文原理 |
| **设计语言** | 国风底蕴（水墨/传统元素）+ 天文科技（发光线条/数据可视化）融合风格 |

### 1.4 目标用户

| 用户类型 | 特征 | 需求 |
|---------|------|------|
| **核心用户** | 10~18岁青少年 | 通过趣味互动学习节气科学知识，易于理解的科普内容 |
| **评审专家** | 比赛评委 | 科学性正确、原创性突出、专业性扎实、普及性强 |
| **次要用户** | 科技馆/文化馆参观者、传统文化爱好者 | 沉浸式体验节气文化与科学 |

## 2. 核心功能

### 2.1 交互方式（由主到次）

| 优先级 | 交互方式 | 实现逻辑 | 适用场景 |
|--------|---------|---------|---------|
| **主交互** | 手势识别 | TensorFlow.js Handpose检测手部21关键点，4种手势映射 | 支持摄像头的设备 |
| **降级1** | 点击节气卡片 | 24节气网格，点击进入详情 | 用户拒绝摄像头权限 |
| **降级2** | 左右滑动黄道带 | 触屏滑动横向滚动时间轴 | 手机端主流操作 |
| **降级3** | 前后翻页 | 上一页/下一页按钮导航 | 所有设备通用 |
| **降级4** | 摇一摇 | DeviceMotion API监听加速度变化 | 趣味探索模式 |
| **降级5** | 日期选择器 | HTML5日期选择器展示全年 | PC端用户 |
| **降级6** | 拖拽交互 | 拖拽节气图标到"地球"动画上 | 互动感最强的降级方案 |
| **降级7** | 语音唤醒（可选） | Web Speech API识别节气名称 | 高阶降级方案 |

### 2.2 手势-节气映射

| 手势 | 识别逻辑 | 唤醒节气 | 动画/音效 |
|------|---------|---------|----------|
| 握拳 | 指尖与掌心距离小于阈值 | **立春**（万物起始） | 种子破土动画 + 春雷声 |
| 手掌张开 | 五指伸直展开 | **夏至**（太阳直射北回归线） | 烈日辉光 + 蝉鸣 |
| 剪刀手（食指中指伸出） | 仅指尖3和指尖4有效 | **秋分**（昼夜平分） | 天平平衡动画 + 秋风声 |
| 挥手（手掌左右摆动） | 手腕关键点横向位移 | 激活"全年节气预览"模式 | 横向滑动时间轴 |

> 说明：原方案中的OK手势（冬至）因移动端前置摄像头识别难度较高，暂不实现，冬至改为其余手势或点击触发。

### 2.3 功能模块

| 模块 | 功能描述 |
|------|---------|
| **动态黄道带** | 主视觉展示太阳黄经0°~360°，标注24节气精确位置，用户可交互拖拽/滑动 |
| **节气详情页（双标签）** | 每个节气含【科学】和【民俗】两个标签页，一键切换：<br>- 科学：太阳黄经度数、直射点纬度、昼夜时长、圭表影长<br>- 民俗：物候现象、农事活动、传统习俗 |
| **科学实验室** | 独立子页面，含2个互动模块：<br>1. 圭表测影：用户滑动改变日期，影长动态变化（夏至最短、冬至最长）<br>2. 日晷模拟：展示不同节气太阳高度角变化 |
| **实时校准** | 调用公历日期，显示今天处于哪个节气、距离下一个节气还有多少黄经角度 |
| **加载场景** | 显示"正在加载手势识别引擎"进度条，Handpose模型预加载 |

## 3. 核心流程

### 3.1 主流程

```
用户进入页面
    │
    ▼
加载场景（预加载Handpose模型 + 节气数据）
    │
    ▼
摄像头权限检测
    ├── 已授权 → 开启手势识别模式
    │       │
    │       ▼
    │   动态黄道带主界面（24节气概览）
    │       │
    │       ├── 握拳 → 立春详情（科学/民俗双标签）
    │       ├── 手掌张开 → 夏至详情
    │       ├── 剪刀手 → 秋分详情
    │       ├── 挥手 → 全年节气滑动预览
    │       └── 其他 → 点击/滑动节气位置进入详情
    │
    └── 拒绝/不可用 → 自动切换到降级交互模式
            │
            ▼
        节气卡片网格（点击进入）/ 黄道带滑动浏览
            │
            ▼
        摇一摇随机唤醒 / 日期选择器 / 拖拽
```

### 3.2 节气详情页流程

```
进入节气详情
    │
    ▼
默认显示【科学】标签
    ├── 太阳黄经度数（例：春分 0°）
    ├── 太阳直射点纬度动画
    ├── 昼夜时长对比
    └── 圭表影长示意图
    │
    └── 一键切换【民俗】标签
        ├── 物候现象（古文+白话解释）
        ├── 农事活动
        └── 传统习俗
    │
    ▼
返回黄道带主界面
```

### 3.3 科学实验室流程

```
进入科学实验室
    │
    ├── 圭表测影互动
    │       └── 滑动滑块改变日期
    │           ├── 圭表影长动态变化
    │           ├── 标注夏至（影最短）/ 冬至（影最长）
    │           └── 文字解释天文原理
    │
    └── 日晷模拟
            └── 选择节气
                ├── 太阳高度角动画
                └── 日晷指针阴影变化
```

## 4. 用户界面设计

### 4.1 设计系统定位

**风格定位**：「国风科技」—— 以 Academia（学院风）为传统文化基底 × Modern Dark（现代深色模式）为科技视觉骨架，融合 Chinese Simplified 中文字体生态。

本设计系统基于项目技能库 (`ui-ux-pro-max-skill`) 中的设计数据集构建，所有色彩、字体、间距值均来自数据驱动的设计令牌体系，非泛化描述。

---

### 4.2 设计令牌体系 (Design Tokens)

#### 4.2.1 色彩系统

参考 `colors.csv` 中 Educational App（教育类，色号#9）配色模式，结合 Academia 风格的深色基底 + 中国传统文化色彩提炼：

| 令牌名 | 色值 | 用途 | 来源参考 |
|--------|------|------|---------|

| `bg-elevated` | `#141a2e` | 卡片/面板表面 | 深色模式 elevated surface |
| `bg-warm` | `#1c1714` | 民俗标签页背景（仿古宣纸深色版） | Academia Mahogany `#1C1714` |
| `foreground-primary` | `#ededeF` | 主文字色（科学标签） | Modern Dark foreground |
| `foreground-warm` | `#e8dfd4` | 主文字色（民俗标签） | Academia Parchment `#E8DFD4` |

| 层级 | 字体 | 字重 | 字号 | 行高 | 用途 |
|------|------|------|------|------|------|

| **科学数据标题** | `Exo`（科技无衬线） | 600 (SemiBold) | 20px-28px | 1.3 | 科学标签页标题 |

### 4.3 页面级设计规范

#### 4.3.1 加载页

| 要素 | 规范 |
|------|------|
| **背景** | `bg-deep` (#0a0e1a)，无图案纯色 |
| **加载指示器** | 环形进度环，使用 `accent-gold` (#d4a574) 描边，参考 Modern Dark 加载模式 |
| **文字** | `Noto Sans SC` 400 14px `muted-ink`，"正在加载手势识别引擎…" |
| **动效** | 环形旋转（infinite spin, 2s linear），无粒子/水墨等装饰性动画 |
| **手势模型进度** | 文本百分比提示（"已加载 67%"），进度环实时填充 |
| **原则** | 快速实用，无装饰性加载动画（参考 Quick Reference §7：motion-meaning，每帧动画必须有因果关系） |

#### 4.3.2 黄道带主页面（核心场景）

| 要素 | 规范 |
|------|------|
| **背景** | `bg-deep` (#0a0e1a)，全屏覆盖 |
| **黄道带** | 半透明弧形轨道，使用 `accent-gold` (#d4a574) 描边，1.5px 线条。0°~360° 刻度标注 |
| **节气标记** | 24个星点分布在黄道带上，当前节气使用 `accent-red` (#c23b22) 高亮发光，其余使用 `accent-gold` 半透明 |
| **当前状态栏** | 顶部固定区域，显示"今日节气：XX | 黄经：XXX° | 距下个节气：X天"，`JetBrains Mono` 12px `muted-ink` |
| **操作区** | 底部 20% 区域预留手势操作提示（文字 + 图标组合），非装饰性 |
| **交互** | 点击/滑动节气标记 → 进入详情；握拳手势 → 唤醒立春；张手 → 夏至 |
| **原则** | 信息密度适中，黄道带为主视觉焦点，无漂浮粒子/云纹等装饰（参考 Quick Reference §4：style-match，每元素有功能目的） |

### 4.4 交互反馈规范

参考 `ui-ux-pro-max-skill` Quick Reference §2 触控与交互标准：

| 交互 | 规范 |
|------|------|
| **触摸目标尺寸** | 最小 44×44px，所有可点击元素满足此标准（QR §2 touch-target-size） |
| **点击反馈** | 按钮/卡片按下时 scale 0.97 + 背景色微变，200ms 内完成（QR §7 duration-timing） |
| **手势过渡** | 手势识别成功后，300ms 内完成页面/动画切换，使用 ease-out 缓动 |
| **加载状态** | 手势模型加载 >300ms 时显示进度条；操作 >1s 显示骨架屏（QR §7 loading-states） |
| **禁用状态** | 模型未就绪时手势区域显示 "⌛ 准备中" 文字提示，降低透明度至 0.38（QR §8 disabled-states） |
| **错误状态** | 手势识别失败时，震动反馈（DeviceMotion API 模拟）+ 文字提示"请调整手势重试" |
| **动画原则** | 仅对状态变化（进入详情/切换标签/数据更新）使用动画，不对装饰元素使用动画（QR §7 motion-meaning） |

### 4.7 暗色/亮色模式

| 模式 | 策略 |
|------|------|
| **科学标签（默认暗色）** | `bg-deep` → `foreground-primary` 高对比，数据卡片使用 `surface-glass` 半透明底 |
