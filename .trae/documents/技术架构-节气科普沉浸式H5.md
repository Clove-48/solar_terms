# 技术架构文档：二十四节气·天文科普沉浸式H5

> 项目名称：**循时 —— 二十四节气漫游**
> 最后更新：2026-06-14 | 状态：Phase 5 验收中

## 1. 项目概述

一款基于原生前端技术栈的轻量级 H5 应用，以**太阳黄经**为科学主线，通过触控交互沉浸式探索二十四节气。所有计算在浏览器本地完成，无后端依赖。

### 2.2 架构原则

- **单一职责**：每个模块只负责一个功能领域
- **可用性优先**：基础交互（点击/滑动）始终可用，保证核心内容浏览
- **无状态设计**：每个页面/组件内部管理自身状态，避免全局状态耦合

## 3. 技术栈选型详解

| 层级 | 技术选型 | 选型理由 |
|------|---------|---------|
| **前端框架** | 原生 HTML/CSS/JS | 零依赖、轻量（< 50KB gzip）、无构建体积膨胀 |
| **构建工具** | Vite 6 (vanilla JS) | 极速 HMR、原生 ESM 支持、Tree Shaking |
| **Canvas 引擎** | 原生 Canvas 2D API | 黄道带动画无需 WebGL 级复杂度，2D API 足够 |
| **音频** | Web Audio API + AudioContext | 低延迟音效播放，程序化合成（无需音频文件） |
| **触屏事件** | Pointer Events + Touch Events | 统一鼠标/触屏/手写笔事件处理 |
| **设备传感** | DeviceMotion API | 摇一摇交互所需 |
| **样式** | CSS Custom Properties + CSS Grid/Flexbox | 运行时变量切换暗色/暖色模式 |

## 5. 核心模块设计

### 5.1 黄道带 Canvas 渲染 (`zodiacCanvas.js`)

**国风科技重设计要素：**
- 深空径向渐变背景 + 80颗闪烁星点
- 金色渐变黄道带轨道 + 外层发光辉光
- 28星宿微点环绕黄道带（古籍星图元素）
- 四象标注（青龙/朱雀/白虎/玄武）于四正点（春分/夏至/秋分/冬至）
- 三级刻度线（15°细线 / 45°中粗 / 90°朱砂红加粗）
- 当前节气：三层辉光 + 朱砂红圆点 + 金色边框 + 大字名称
- 中心显示当前太阳黄经度数值
- 底部装饰文字 "二 十 四 节 气 · 黄 道 观 测"

### 5.3 场景引擎 (`sceneEngine.js`)

动态粒子动画系统，根据节气类型自动匹配场景：
- 春季节气 → 花瓣飘落（粉/白色粒子 + 诗人剪影）
- 夏季节气 → 烈日辉光（径向渐变 + 热浪波纹）
- 秋季节气 → 落叶纷飞（金黄/橙色粒子 + 月亮渐显）
- 冬季节气 → 雪花飘落（白色粒子 + 冰晶效果）

### 5.4 圭表测影计算模型 (`gnomon.js`)

```
太阳赤纬 δ = arcsin(sin(λ) × sin(23.44°))
正午太阳高度角 H = 90° - |34° - δ|     (观测点：河南登封 34°N)
圭表影长 L = 8尺 × cot(H)              (表高 = 8尺 ≈ 2.67m)
```

### 5.5 程序化音效系统 (`soundEffects.js`)

所有音效通过 Web Audio API 实时合成，无需预置音频文件：
- 立春：春雷声（低频振荡 + 随机爆裂噪声）
- 夏至：蝉鸣（高频振荡 + LFO 调频颤音）
- 秋分：秋风（带通滤波器白噪声扫频）
- 冬至：寒风 + 雪声（低通滤波器白噪声）

### 5.6 交互辅助系统 (`fallbackSystem.js`)

统一切换多种交互模式：

- **前后翻页按钮**：所有设备通用，切换上一个/下一个节气
- **摇一摇**：支持 DeviceMotion 的设备，随机跳转到某个节气
- **拖拽**：桌面端黄道带页面，拖拽节气卡片到场景区触发详情
- **日期选择器**：所有页面可用，按日期定位节气

## 6. 数据流设计

### 6.1 页面切换数据流

```
用户操作 → 交互层（点击/滑动/翻页）
  → 解析出目标节气 ID + 动作类型
  → Store 更新（currentTermId, viewMode）
  → 路由切换（hash 变化 → #/detail/1）
  → 目标页面渲染 + CSS 动画触发
```

### 6.2 实时校准数据流

```
获取当前日期 → calendar.js 计算太阳黄经（插值法）
  → 匹配 solarTerms.json 中的节气区间
  → payload: { currentTerm, nextTerm, longitude, daysUntilNext }
  → Store 更新 → 顶部状态栏渲染（JetBrains Mono 等宽字体）
```

### 6.3 科学实验室数据流

```
用户滑动日期滑块
  → gnomon.js 计算影长 + sunPosition.js 计算直射点/昼长
  → gnomonCanvas.js 更新圭表渲染（表杆 + 影子 + 太阳射线）
  → 数据卡片实时更新（影长/高度角/赤纬/昼长/直射点/昼夜比）
  → 文字解说模块动态更新对应文案
```

## 7. 页面架构

### 7.1 页面渲染模式

所有页面采用 **logic.js 合一模式**：HTML 模板 + CSS 样式 + JS 逻辑均在单个 logic.js 文件中，通过 `container.innerHTML` 注入 DOM，避免文件碎片化。

```
src/pages/{pageName}/logic.js
  ├── render()        → 注入 HTML + <style>
  ├── onEnter(params) → 页面进入回调（数据加载、渲染触发）
  ├── onLeave()       → 页面离开回调（清理监听、释放资源）
  └── 交互逻辑        → 事件绑定、动画控制
```

### 7.2 路由设计

| 路由 | 页面 | 说明 |
|------|------|------|
| `#/loading` | 加载页 | 加载节气数据 + 进度环，完成后自动跳转 |
| `#/zodiac` | 黄道带主页 | 核心交互场景，Canvas 渲染 |
| `#/detail/{id}` | 节气详情 | 科学/民俗双标签，id 对应节气ID(1-24) |
| `#/lab` | 科学实验室 | 圭表测影 + 日晷模拟 |

## 11. 关键设计决策记录 (ADR)

| 决策 | 选择 | 放弃方案 | 理由 |
|------|------|---------|------|
| Canvas vs SVG | Canvas 2D | SVG、Three.js | 黄道带交互不需 3D，Canvas 性能优于大量 DOM 元素的 SVG |
| 无框架 | 原生 JS | React、Vue | 页面级场景 < 6 个，框架开销 > 收益 |
| CSS 变量 | CSS Custom Properties | Sass/Less 变量 | 运行时切换模式需要，CSS 变量天然支持 |
| 页面模式 | logic.js 合一 | 拆分 HTML/CSS/JS | 减少文件碎片化，单文件更易维护 |
| 音效方案 | Web Audio API 程序合成 | 预置 OGG/MP3 文件 | 零外部依赖，减小构建体积 |
